自幹窮人服務 - 架設nstube取代ngrok的服務

發表於2024-01-23更新於2024-01-24

利用NGINX, OpenSSH和certbot建構如同ngrok的服務

開發網路服務時為了能在內部測試, 通常會利用ngrok打通內網與外部網路的溝通. 但免費的ngrok服務每次執行都會隨機配發網址, 這小小的缺點會造成開發某些服務時會造成困擾, 尤其是採用第三方的OAuth2認證流程與Callback URL等服務.
基於勤儉持家理由,利用數個Linux軟體建構如同ngrok的服務.

目標 建立nstube (Nginx & SSH Tunnel)容器

利用docker技術封裝Nginx, OpenSSH, certbot, DDNS 等程式建構nstube服務,免費達成取代ngrok的目的
功能包含:

  1. 免費的Domain Name - duckdns 或 No-IP
  2. 免費的HTTPS憑證 - Let’s Encrypt
  3. 依據subdomain或URL prefix 分流多種不同的服務, 例如:
    https://dev.foo.duckdns.org –> 第一個服務
    https://www.foo.ducknds.org –> 第二個服務
    https://shop.foo.duckdns.org/admin –> 第三種服務

背景知識

  1. 設定Firewall/NAT與Port Forwarding
  2. 浮動IP與申請DDNS服務
  3. 設定NGINX與reverse proxy
  4. 利用certbot申請免費的Let’s Encrypt憑證
  5. 設定crontab,定期更新DDNS與Let’s Encrypt憑證
  6. 設定OpenSSH與SSH reverse tunnel
  7. docker和container

架構圖

一般情況, 一個container負責一種任務/服務, 使其易於擴展與維護; 而本篇利用容器特性把數個軟體包裝成一個容器, 目的是為了方便未來佈署與設定, 另一個重點是不想因為安裝軟體而污染目前的Linux系統開發環境


重點說明:

  1. 修改對外路由器/IP分享器的WAN/firewall設定, 指定TCP port 80, 443 到內部跑nstube的主機
  2. 申請DDNS服務 - 現此時, 個人覺得 duckdns.org 服務容易申請也蠻好用的.
  3. 修改NGINX設定(nginx.conf), 設定DDNS取得的網域, 再把HTTP 80服務開設起來並設定好certbot需要的路徑/.well-known/acme-challenge/
  4. 利用certbot取得HTTPS憑證(在這裡不用DNS TXT record模式)
  5. 再次修改nginx.conf 讓它支援HTTPS連線服務
  6. 利用NGINX location { reverse proxy } 與 SSH reverse tunnel 建立內部網路服務與外部連線的通道
